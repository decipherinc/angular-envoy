EnvoyFormController
controller in module fv.envoy

Description

EnvoyFormController replaces FormController with itself; any time you use the form directive, your controller will be this instead, except if your form has no name attribute, at which point it is ignored by Envoy.

Methods

  • $setValidity(token, value, control)

    If the number of errors in this form has increased or decreased and the control being set valid or invalid is a member of this form proper, then tell $envoy handle the change.

    Note: we only tell $envoy to update if the control is a direct descendant of this form.

    Parameters
    ParamTypeDetails

    Validation token

    valueboolean*

    If truthy, then the validation token is in an error state.

Properties

  • $alias

    If the parent form directive contains an "alias" attribute, we'll use it to look up messages. This is useful if your form name is "dynamic" (interpolated). Note: interpolated form names were not implemented before AngularJS 1.3.0.

    Defaults to whatever the name of the form is.

    If the alias is not present in this form's Scope, then it is placed there--much like FormController places its name attribute on its Scope, if present. Because collisions could exist in the case of "dynamic" forms, the form directive must create a new Scope.

  • $isForm

    This will always be true for any form touched by Envoy. The reason for its existence is simply that it can be practically difficult to tell the difference between a FormController and an NgModelController.
